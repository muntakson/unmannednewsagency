# 무인 뉴스 에이전시 만들기

Paperclip을 사용해서 AI 에이전트가 24시간 자동으로 뉴스를 취재하고, 기사를 쓰고, 편집하고, 발행하는 **무인 뉴스 에이전시**를 만드는 가이드입니다.

실제 운영 중인 두 가지 사례를 참고합니다:
- **ESP32 Times** (`esptimes.iotok.org`) — ESP32 마이크로컨트롤러 뉴스
- **Approtech Times** (`appro.iotok.org`) — 적정기술 뉴스

## 1. 개념 이해

### Paperclip이란?

Paperclip은 AI 에이전트 오케스트레이션 플랫폼입니다. 쉽게 말해 **AI 직원들로 회사를 만드는 시스템**입니다.

- **회사(Company)** = 뉴스 에이전시 (예: "테크 타임즈")
- **에이전트(Agent)** = AI 직원 (Scout, Writer, Editor, CEO)
- **이슈(Issue)** = 할 일 / 기사 (태스크 보드의 카드 하나)
- **하트비트(Heartbeat)** = 에이전트가 깨어나서 일하는 주기

### 기사가 만들어지는 과정

```
Scout (취재기자)          Writer (작가)           Editor (편집자)
    |                        |                       |
    | 1. 웹 검색으로          | 3. Scout의 브리프를     | 5. 기사 품질 확인
    |    뉴스 발견            |    읽고 기사 작성       |    (정확성, 문체)
    |                        |                       |
    | 2. Writer에게           | 4. 기사를 이슈에        | 6. 승인 → done(발행)
    |    태스크 생성           |    저장, 편집 요청      |    또는 피드백
    v                        v                       v

[todo] ──→ [in_progress] ──→ [in_review] ──→ [done] = 웹사이트에 게시!
```

핵심: **기사 = 이슈(Issue)**입니다. 별도 데이터베이스가 필요 없습니다. Paperclip의 이슈 시스템이 곧 CMS입니다.

### 어댑터(Adapter) 선택

에이전트가 사용할 LLM을 선택합니다:

| 어댑터 | 설명 | 비용 | 추천 |
|--------|------|------|------|
| `ollama_local` | 로컬 Ollama (llama3.1 등) | 무료 | 테스트용, GPU 필요 |
| `groq` | Groq 클라우드 API (llama-3.3-70b) | 저렴 (~$2-5/월) | **운영 추천** |
| `claude-local` | Claude Code | 유료 | 고품질 필요시 |

## 3. 회사(Company) 만들기

Paperclip API로 뉴스 에이전시(회사)를 생성합니다.

```bash
API="http://localhost:3100/api"

# 회사 생성
curl -s -X POST "$API/companies" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "내 뉴스 에이전시",
    "description": "AI가 운영하는 OO 분야 뉴스 사이트"
  }'
```

응답에서 `id`를 기록합니다 — 이후 모든 API 호출에 사용됩니다:

```json
{
  "id": "abc123-...",
  "name": "내 뉴스 에이전시"
}
```

```bash
COMPANY_ID="abc123-..."  # 실제 ID로 교체
```

## 4. 에이전트 4명 만들기

뉴스 에이전시에는 4명의 에이전트가 필요합니다:

| 에이전트 | 역할 | 하는 일 |
|----------|------|---------|
| **CEO** | 편집장 | 편집 방향 설정, 팀 조율 |
| **Scout** | 취재기자 | 뉴스 검색, 소스 수집, Writer에게 브리프 전달 |
| **Writer** | 작가 | Scout의 브리프를 바탕으로 기사 작성 |
| **Editor** | 편집자 | 기사 품질 검토, 발행 승인 |

```bash
CEO=$(curl -s -X POST "$API/companies/$COMPANY_ID/agents" \

  -d '{

    "title": "Chief Editor",
    "role": "ceo",
    "icon": "crown",
    "adapterType": "groq",

      "apiKey": "'"$GROQ_API_KEY"'",

        "enabled": true,
```

## 5. 시스템 프롬프트 작성법

시스템 프롬프트는 에이전트의 **두뇌**입니다. 잘 쓰면 기사 품질이 올라가고, 못 쓰면 엉뚱한 결과가 나옵니다.

### Scout 프롬프트 (핵심)

Scout의 프롬프트에는 반드시 다음이 포함되어야 합니다:

```
당신은 OO뉴스의 취재기자입니다.

워크플로우:
1. list_my_issues()를 호출해서 할당된 태스크 확인
2. 태스크가 있으면 해당 주제를 조사
3. 태스크가 없으면 자율적으로 뉴스 검색

뉴스 소스:
- [소스1 URL] (설명)
- [소스2 URL] (설명)
- web_search()로 키워드 검색: "OO 기술", "OO 뉴스" 등

각 뉴스를 발견하면:
1. fetch_url()로 기사 전문과 og:image를 가져옴
2. 핵심 사실, 소스 URL, 이미지를 포함한 리서치 브리프 작성
3. create_issue()로 Writer에게 태스크 생성 (assignee_agent_id 포함)
4. 자기 태스크를 done으로 변경

반드시 Source: [제목](url)과 이미지 출처를 브리프에 포함하세요.
```

**중요**: Scout에게 **구체적인 뉴스 소스 URL**을 알려줘야 합니다. "뉴스를 찾아라"만으로는 부족합니다.

### Writer 프롬프트

```
당신은 OO뉴스의 기자입니다. 일반 독자가 이해할 수 있는 명확한 기사를 작성합니다.

워크플로우:
1. list_my_issues()로 할당된 태스크 확인
2. 가장 우선순위 높은 todo 태스크 선택
3. update_issue()로 상태를 in_progress로 변경
4. Scout의 리서치 브리프를 읽고 기사 작성
5. update_issue()로 이슈 description에 기사 저장
6. 상태를 done으로 변경

기사 형식:
- 600-1200 단어
- ## 헤더로 섹션 구분
- 마크다운 형식
- Scout 브리프의 Source: [제목](url)을 반드시 보존
- ![이미지](url)과 *Image: [출처](url)* 보존
```

**참고**: Writer에게는 `web_search`/`fetch_url` 도구가 **없습니다**. Scout의 브리프에 있는 정보만으로 기사를 씁니다. 따라서 Scout의 브리프 품질이 곧 기사 품질입니다.

### Editor 프롬프트

```
당신은 OO뉴스의 편집자입니다. 기사의 품질을 검토합니다.

워크플로우:
1. list_my_issues()와 list_company_issues(status='in_review')를 호출
2. in_review 상태의 기사를 검토
3. 확인 항목: 정확성, 명확성, 완결성, 소스 표기, 이미지 출처
4. 좋으면 상태를 done으로 변경 (발행!)
5. 수정 필요하면 코멘트로 피드백 후 Writer에게 재할당
```

### 프롬프트 작성 팁

| 좋은 예 | 나쁜 예 |
|---------|---------|
| "fetch_url()로 해당 URL의 og:image를 추출하세요" | "이미지를 찾으세요" |
| "create_issue()로 Writer에게 태스크를 만드세요" | "Writer에게 전달하세요" |
| "Source: [제목](url) 형식을 반드시 포함" | "출처를 밝히세요" |
| "상태를 done으로 변경하세요" | "완료 처리하세요" |

핵심: **어떤 도구를**, **어떤 형식으로** 사용할지 구체적으로 지시하세요.

### Temperature 설정

| 에이전트 | Temperature | 이유 |
|----------|-------------|------|
| Scout | 0.8 | 창의적 주제 발견 |
| Writer | 0.6 | 균형잡힌 글쓰기 |
| Editor | 0.4 | 보수적, 신중한 검토 |
| CEO | 0.7 | 전략적 판단 |

## 6. 하트비트(자동 실행) 설정

하트비트는 에이전트가 **자동으로 깨어나서 일하는 주기**입니다.

### 세 가지 깨우기 방식

| 방식 | 언제 | 예시 |
|------|------|------|
| **타이머** | `intervalSec`마다 자동 | Scout가 12시간마다 뉴스 검색 |
| **할당 감지** | 이슈가 에이전트에게 할당될 때 | Scout가 Writer에게 태스크 생성 → Writer 30초 내 깨어남 |
| **수동 깨우기** | 관리자가 직접 | 대시보드에서 "Wake" 버튼 클릭 |

### 추천 하트비트 설정

```
CEO:    43200초 (12시간) — 하루 2회 전략 검토
Scout:  43200초 (12시간) — 하루 2회 뉴스 탐색
Writer: 43200초 (12시간) — 하루 2회 (+ 할당 시 즉시 깨어남)
Editor: 비활성화 — llama 모델의 tool call 불안정 이슈
```

**실전 교훈**: llama-3.3-70b 모델은 Editor의 tool call에서 파라미터를 `"none"`으로 출력하는 버그가 있습니다. 그래서 Approtech Times에서는 Editor 하트비트를 꺼두고, Writer가 직접 `done`으로 상태를 변경합니다.

**파이프라인 단순화**:
```
정상 흐름:  Scout → Writer → Editor → done
실전 흐름:  Scout → Writer → done  (Editor 생략)
```

## 7. 뉴스 웹사이트 만들기

뉴스 웹사이트는 **Paperclip API에서 이슈를 읽어와서 기사로 보여주는** Express.js 서버입니다.

### 기본 구조

```
mynews/
├── server.js          # Express 서버 (메인)
├── public/
│   └── styles.css     # CSS 스타일
├── package.json
└── node_modules/
```

### server.js 핵심 코드

```javascript
import express from "express";
import { marked } from "marked";
import path from "path";
import { fileURLToPath } from "url";

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const app = express();
const PORT = process.env.PORT || 3200;
const PAPERCLIP_API = process.env.PAPERCLIP_API || "http://127.0.0.1:3100/api";
const COMPANY_ID = process.env.COMPANY_ID || "";

if (!COMPANY_ID) {
  console.error("ERROR: COMPANY_ID 환경변수가 필요합니다.");
  process.exit(1);
}

app.use(express.static(path.join(__dirname, "public")));

// ── Paperclip API에서 데이터 가져오기 ──

async function fetchIssues(status) {
  const url = `${PAPERCLIP_API}/companies/${COMPANY_ID}/issues?status=${status}&limit=50`;
  const res = await fetch(url);
  if (!res.ok) return [];
  return res.json();
}

async function fetchIssue(id) {
  const res = await fetch(`${PAPERCLIP_API}/issues/${id}`);
  if (!res.ok) return null;
  return res.json();
}

async function fetchAgents() {
  const res = await fetch(`${PAPERCLIP_API}/companies/${COMPANY_ID}/agents`);
  if (!res.ok) return [];
  return res.json();
}

// ── 기사 필터링 ──
// 이슈 중 기사에 해당하는 것만 골라냅니다

async function getArticles() {
  const [done, inReview, inProgress] = await Promise.all([
    fetchIssues("done"),
    fetchIssues("in_review"),
    fetchIssues("in_progress"),
  ]);

  const isArticle = (issue) =>
    issue.description?.length > 200;  // 본문 200자 이상이면 기사

  return {
    published: done.filter(isArticle),       // 발행된 기사
    drafts: [...inReview, ...inProgress].filter(isArticle),  // 작성 중
  };
}

// ── 마크다운에서 이미지/소스 추출 ──

function extractImage(desc) {
  // ![alt](url) 패턴에서 이미지 URL 추출
  const match = desc.match(/!\[[^\]]*\]\(([^)]+)\)/);
  return match ? match[1] : null;
}

function extractSource(desc) {
  // Source: [제목](url) 패턴 추출
  const match = desc.match(/Source:\s*\[([^\]]+)\]\(([^)]+)\)/i);
  return match ? { title: match[1], url: match[2] } : null;
}

function cleanTitle(title) {
  // "Write: 기사 제목" → "기사 제목"
  return title.replace(/^(Write|Scout|CEO|Editor):\s*/i, "").trim();
}

// ── 라우트 ──

// 홈페이지
app.get("/", async (req, res) => {
  const { published } = await getArticles();
  const agents = await fetchAgents();

  const articleCards = published.map(article => {
    const image = extractImage(article.description || "");
    const title = cleanTitle(article.title);
    return `
      <article class="card">
        ${image ? `<img src="${image}" alt="${title}">` : ""}
        <h2><a href="/article/${article.id}">${title}</a></h2>
```

## 9. 대시보드로 기사 관리하기

대시보드(`/dashboard`)에서 수동으로 기사를 만들 수 있습니다.

### 기사 생성 시 입력 품질이 결과를 결정합니다

| 입력 방법 | 결과 |
|-----------|------|
| 구글 검색 URL | **나쁨** — JS 렌더링된 구글 결과 페이지는 파싱 불가 |
| 직접 기사 URL | **좋음** — Scout가 실제 내용을 추출해서 상세한 브리프 작성 |
| 짧은 주제 텍스트 | **보통** — Scout가 web_search 사용하지만 결과가 불안정 |
| 주제 + 핵심 사실 직접 입력 | **최고** — Writer가 사실 기반으로 작성, 환각 없음 |

**최선의 방법**: 직접 기사 URL을 붙여넣거나, 이슈 설명에 핵심 사실을 직접 적어주세요.

### 대시보드 API 프록시

뉴스 서버에 관리용 API를 추가합니다:

```javascript
app.use(express.json());

// 이슈(기사) 생성

  const result = await fetch(`${PAPERCLIP_API}/companies/${COMPANY_ID}/issues`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(req.body),
  });
  res.json(await result.json());
});

// 상태 변경
app.post("/dashboard/api/issues/:id/status", async (req, res) => {

    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ status: req.body.status }),
  });
  res.json(await result.json());
});

// 에이전트 수동 깨우기
app.post("/dashboard/api/agents/:id/wakeup", async (req, res) => {
  const result = await fetch(`${PAPERCLIP_API}/agents/${req.params.id}/wakeup`, {
    method: "POST",
  });
```

## 11. 운영 팁과 주의사항

### 자주 하는 실수

| 실수 | 해결 |
|------|------|
| `assigneeId`로 이슈 할당 | `assigneeAgentId`를 사용하세요 |
| 에이전트 config 부분 업데이트 | 전체 config를 읽어서 수정 후 저장 (API 키 유실 방지) |
| 서버 재시작 시 포트 충돌 | `pkill -f "tsx"` 후 재시작 |
| Ollama에서 tool call 안 됨 | `stream: false` 설정 필수 |
| Google URL을 Scout에게 전달 | 직접 기사 URL을 사용하세요 |
| Facebook 페이지를 fetch_url | Facebook은 스크래핑이 어려움 |

### 에이전트 수동 깨우기

```bash
# 특정 에이전트 깨우기
curl -X POST http://localhost:3100/api/agents/$SCOUT_ID/wakeup

# 하트비트 실행 기록 확인
curl http://localhost:3100/api/companies/$COMPANY_ID/heartbeat-runs
```

### 에이전트 설정 변경

```bash
# 에이전트 정보 수정 (회사 prefix 없이!)
curl -X PATCH "http://localhost:3100/api/agents/$SCOUT_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "adapterConfig": { ... 전체 config ... }
  }'
```

**주의**: `adapterConfig`를 부분 업데이트하면 기존 값(API 키 등)이 사라집니다. 반드시 현재 config를 먼저 읽고, 수정한 전체 config를 보내세요.

### Groq 429 에러 (Rate Limit)

Groq API는 무료 티어에서 분당 요청 수 제한이 있습니다. Paperclip의 Groq 어댑터는 자동으로 최대 3회 재시도합니다.
